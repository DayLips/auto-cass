# Auto-Cass

Скрипт generation-script-files создает csv файлы в папке data. Сами csv файлы - это файлы чеков одной каасы {номер магазина}_{номер кассы}.csv
Скрипт put-script считывает csv файлы в папке data и записывает их данные в базу данных store_receipts, потом удаляет csv файлы. Помимо этого он создает таблицы в базе дынных(если их нет),
заполняет таблицы categories, products, shops, casses даными - имитируя реальную бд с данными, которые один раз положили и больше не трогают. Логично ведь название магазина редко меняется.

## 🚀 Возможности
- Генерация тестовых данных чеков с товарами, категориями, скидками
- Автоматическое создание таблиц в PostgreSQL (если их нет)
- Заполнение справочных таблиц (категории, товары, магазины, кассы)
- Чтение CSV-файлов из папки `data` и загрузка в БД
- Очистка папки `data` после успешной загрузки

## 📦 Установка

1. Клонируйте репозиторий:   
   ----------------------------------------------------
   git clone https://github.com/DayLips/auto-cass   
   cd auto-cass/put-script
   
   ----------------------------------------------------
3. Работа с generation-script-files:   
   В общем ничего не надо, используются встроенные библиотеки в Python. Если хотит поменять примеры продуктов и категорий, то они находятся в tools/data_to_gen.py в словаре.
   Смотрите только если вы поменяли их, то надо будет поменять их в скрипте put-script, а то при первом запуске всорого скрипта произойдут ошибки и данные не занесутся в бд.
4. Работа с put-script:

   Для начала вам надо перейти в директорию put-script/ и создать там виртуальное окружение, затем активировать его и закачать библиотеки из файла requiremnts.txt
   
   -------------------------------
   cd put-script   
   python -m venv venv   
   venv/Scripts/activate   
   pip install -r requirements.txt
   
   -------------------------------   
   Также вам надо будет создать .env файл и записать в него одну строчку: DATABASE_URL=postgresql://{Логин}:{Пароль}@{Адрес}:{Хост}/{Название бд}.
   Это параметры подключения к вашей созданной бд. Следующий кстати пункт об этом.
6. Создайте бд:   
   Можете локальную или еще где нибудь. Через psql или dbeaver как вам удобнее. Главное не забудьте пароль:)
7. Планировщик:
   
   Последнее что нужно сделать, это создать две задачи в планировщике задач Windows. Главное первый кто должен выполняться - это скрипт generation-script-files, потом уже put-script.
   Я поставил с разницей в 5 минут. Если вам не надо имитировать чеки (как у меня), то просто либо сами запускайте скрипт put-script. Каждый скрипт запускается при помощи main.py файла.
    
   --------------
   python main.py
   
   --------------
   generation-script-files можно запускать без активированного окружения, а вот put-script только из активированного окружения!!! Не забудьте эти нюансы при создании задачи.

🛠️ Технологии
- Python 3.10+
- SQLAlchemy (ORM)
- PostgreSQL
- python-dotenv, psycopg2-binary
